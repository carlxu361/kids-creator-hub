# 第6关 🎯 Scratch进阶：狼人杀

## 🎯 任务目标
用Scratch做一个狼人杀游戏！你之前已经用Codex尝试过，现在用Scratch来实现。

## ⏰ 预计时间
2-3个周末

---

## 📖 什么是狼人杀？

狼人杀是一个经典的桌游：
- **村民阵营**：找出谁是狼人并投票处决
- **狼人阵营**：隐藏身份，每晚吃掉一个村民
- **特殊角色**：预言家（查验身份）、女巫（救人/毒人）、猎人（死后带走一人）

---

## 🛠️ 设计思路

### 第1步：画出游戏流程图（重要！）

在开始编程前，先用纸笔画出游戏怎么运行：

```
游戏开始
    ↓
分配角色（谁是狼人、谁是村民）
    ↓
[天黑] 狼人选择要吃掉谁
    ↓
[天亮] 公布昨晚谁被吃了
    ↓
玩家讨论，投票选出可疑的人
    ↓
被投票的人出局，公布身份
    ↓
判断游戏是否结束
    ↓
没结束 → 回到"天黑"
结束了 → 显示谁赢了
```

### 第2步：设计角色

需要这些角色：
1. **主持人**（旁白，控制游戏流程）
2. **狼人**（1-2个）
3. **村民**（3-4个）
4. **预言家**（1个）
5. **背景**（村庄、夜晚等）

### 第3步：用Scratch实现

#### 基础版本（第1个周末）

**功能1：角色分配**
```scratch
当绿旗被点击
创建列表"玩家列表"
将"玩家1"加入列表
将"玩家2"加入列表
...（添加所有玩家）

创建列表"角色列表"
将"狼人"加入列表
将"村民"加入列表
...（添加所有角色）

# 打乱角色列表
重复(列表长度)次
    随机交换两个角色的位置
end

# 分配角色给玩家
重复(玩家数量)次
    将角色列表的第[i]项设为玩家[i]的角色
end
```

**功能2：天黑阶段**
```scratch
当收到"天黑"消息
说"天黑了，请闭眼" 2秒
等待2秒
说"狼人请睁眼" 2秒
说"狼人请选择要吃掉的玩家" 2秒
# 这里需要玩家输入选择
```

**功能3：天亮阶段**
```scratch
当收到"天亮"消息
说"天亮了，请睁眼" 2秒
说"昨晚[玩家X]被吃掉了" 2秒
# 显示被吃掉的玩家
```

**功能4：投票阶段**
```scratch
当收到"投票"消息
说"请投票选出你认为的狼人" 2秒
# 收集投票
# 统计票数
# 宣布结果
```

#### 进阶版本（第2-3个周末）

**AI对手：**
让电脑控制一些玩家，能"思考"和"决策"。

**跟AI对话：**
```
我想在Scratch里让电脑控制的玩家能"思考"。
比如，狼人要选择吃掉谁，村民要投票给谁。
你能帮我设计一个简单的AI决策逻辑吗？
要求：
1. 用Scratch的积木块能实现
2. 不要太复杂
3. 有一定的随机性，不要太规律
```

**特殊角色能力：**
- 预言家：每晚可以查验一个人的身份
- 女巫：有一瓶解药和一瓶毒药
- 猎人：死的时候可以带走一个人

## 🧠 设计思考题

1. 怎么让游戏界面更清晰？（用不同的造型表示不同状态）
2. 怎么让AI玩家的行为不像"机器人"？（加入随机性）
3. 怎么判断游戏结束？（狼人全死=村民赢，村民<=狼人=狼人赢）
4. 如果想加"警长"这个角色，代码要怎么改？
